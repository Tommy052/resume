#### Plotly.js를 걷어내고 GPU 기반 시각화 엔진을 직접 만들다

**문제:** 검사 방식에 따라 데이터 규모가 수백 배 차이났다. 소규모 신호 데이터는 Plotly.js로 충분했지만, 대규모 초음파 신호 데이터(수십만 포인트 × 멀티채널)에서는 D3.js + SVG 렌더링조차 심각하게 버벅거렸다. 또한 차트 레이아웃이 3개로 고정되어 검사자가 원하는 방식으로 데이터를 볼 수 없었고, Plotly.js는 커스텀 인터랙션(영역 선택, 오버레이, 다축 동기화) 구현에 한계가 있었다.

**해결:**
- Plotly.js를 완전히 걷어내고 D3.js 기반 시각화 엔진을 자체 설계. 바닐라 JS 프로토타입 → D3 스케일 계산 + React SVG 렌더링 구조로 단계적 리팩토링
- D3.js + SVG로도 대규모 데이터의 버벅거림을 해결할 수 없어, WebGL 렌더링으로 전환. 히트맵 렌더러(`heatmapgpu`), 데이터 히트맵 렌더러(`datheatmapgpu`), 스트립/라인 차트 렌더러(`stripchartgpu`) 3개의 GPU 기반 패키지를 직접 개발
- LOD(Level of Detail) 기반 다운샘플링, 텍스처 타일 분할 렌더링으로 GPU 메모리 한계를 극복
- 검사 데이터의 2D 맵을 3D로 시각화하기 위해 Babylon.js + WebGPU 기반 3D 뷰어 구축
- 좌표계, 스케일, 줌/팬 인터랙션을 추상화하여 검사자가 차트 배치를 자유롭게 구성할 수 있도록 변경

**성과:**
- 고정 3개 레이아웃 → 검사자가 자유롭게 차트를 선택·배치하는 구조로 전환, 검사 유연성 대폭 향상
- 수십만 포인트 데이터를 WebGL로 렌더링하여 60fps 유지 (D3.js + SVG 대비 렌더링 성능 약 10배 향상)
- 2D 히트맵 → Babylon.js + WebGPU 기반 3D 맵 시각화로 확장, 검사 데이터의 입체적 분석 가능
- Plotly.js에서 불가능했던 커스텀 인터랙션(영역 선택, 오버레이, 멀티채널 동기화) 모두 구현
